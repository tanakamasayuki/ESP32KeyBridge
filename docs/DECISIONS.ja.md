# 設計決定の台帳

ゼロベースのユースケース検証([USE_CASES.ja.md](USE_CASES.ja.md))で積み上げた決定の一覧です。正式な仕様は [DATA_MODEL.ja.md](DATA_MODEL.ja.md) / [CORE_DESIGN.ja.md](CORE_DESIGN.ja.md) に反映済みで、この文書は各決定の状態(確定 / 仮置き)と理由を残します。

## 設計原則(確定)

1. **ブリッジは解釈しない。** 文字への解釈は原則ホストの仕事。変換が必要な場面だけ限定的に解釈する。
2. **状態の正本は、その状態を解釈する側が持つ。** ブリッジは解釈しない情報の正本を持たない。
3. **時間を持たない。** 時間依存処理はアダプタ側。core は clock を読まない。
4. **core は移植可能な純粋 C++。** プラットフォーム依存はアダプタ層(= HAL)に閉じる。

## 確定事項

- **Lock 状態の基本**: USB device 出力時の lock の正本は PC。LED report(LockState)の全入力への転送は必須。
- **内部シャドウと正本の連鎖**: 内部 LockState を常に 1 つ保持。lock 報告出力が存在する間はそこが正本(上書き + 転送、自前 toggle なし)。不在の間はブリッジが終端ホストとして press 時 toggle(Caps/Num/Scroll、カナ対象外)。外部正本の出現時はそれが勝ち、消失時は最終値から引き継ぐ。後挿し入力には接続時点の状態を通知。
- **正本の一意性**: 正本 = アクティブな lock 報告出力(常に 1 つ)。v1 では「最初に登録された lock 報告出力」で固定。複数登録は禁止しないが整合保証もしない(1 つを推奨)。**能動 lock 同期は既定で行わない**。
- **v1 スコープ**: キー変換一式 / Lock・LED 一式 / Consumer キー / 文字列→打鍵 / キーボードのレイアウト変換(コア機能)/ マウス(相対のみ)。

## 仮置き(ユースケース検証で矛盾なし。実装で最終確定)

1. 中間表現はキーイベント(press/release)と押下集合。keyboard の値は HID Usage と恒等(UC1〜で検証)。
2. remap は 1 段 lookup・連鎖なし。modifier ビットとキー配列は「ただのキー」に正規化(UC1)。
3. 入力ごとの押下集合 + 和集合統合。release は全入力解放で成立(UC2)。
4. アダプタ責務境界: スキャン・デバウンス・割り当て・時間はアダプタ(UC3)。
5. キー番号空間は HID Usage に限定しない(virtual キー)。表現は実装判断(→ 12 に発展)。
6. 入力の切断・消失時は押下集合をクリアし全 release(UC4)。
7. レイアウト変換 = (キー,Shift)→(キー,Shift) 表 + Shift 抑制・合成。恒等キー素通し。on/off 切替必須。修飾コンボは既定素通し(暫定)。矛盾同時押し非対応(UC5/6/8)。
8. 修飾共有は和集合で自動成立。レイヤー状態は統合後レベルで全入力に効く。物理 Fn は観測不能(UC2 再訪)。
9. Fn 相当 = HID キーを virtual へ remap して消費 + 押下中レイヤー。**置換は press 時確定、release は確定値を離す**(stuck key 防止)。tap/hold は保留。
10. 押下集合は論理押下(物理と 1:1 不要)。トグル・ワンショット・タップ合成はアダプタ(UC7)。
11. レイアウト混在は per-input 変換 + 統合で成立。個別接続では解決不能(UC8)。
12. キーの同一性 = **種別 + 値**(keyboard / consumer / mouse_button / virtual / 将来拡張)。remap は種別またぎ可。出力は表現できる種別のみ送出。データ表現は実装判断(UC9)。
13. 文字列→打鍵: 出力ごとのホストレイアウト記述(文字⇄キー+Shift)。文字単位アトミック、ペーシングはアダプタ、あふれは捨ててカウント、修飾退避/復元 + 遅延オプション。文字は文字のまま出力の縁で展開。制御文字の標準割り当て(実文字のみ、エスケープはパースしない)(UC10)。
14. **2 大分類**: 継続(状態系: 押下集合・和集合・切断クリア・press 時確定)と単発(イベント系: 順序キュー・到着順・アトミック性)。単発 payload = 文字 / キー操作指示(down/up/delay。合成打鍵のみで実キーは通らない)/ 相対値(加算可能)。Consumer は継続(エッジ/連続はホストの解釈)。マウスはボタン継続・移動単発。
15. マウス: ボタン = mouse_button 種別の継続系、移動・ホイール = 相対値。report 合算 + 繰り越し。軸の反転・整数倍率は core。絶対座標は対象外。マウス report はマウス対応出力アダプタを登録した場合のみ。descriptor は enumerate 時確定(UC11)。

## 原理的な不可能性(スコープ外として明記)

1. 出力先 layout に存在しない文字の生成(Alt+テンキー等の OS 固有技は範囲外)。
2. アプリのショートカット束縛(文字ベース / 位置ベース)の自動判別。
3. 修飾状態の多重化(ホストに 1 つしかない)。
4. IME 状態・フォーカスアプリの観測。IME 連携・かな漢字生成。
5. 物理 Fn キーの観測・remap・共有。

## 将来メモ(v1 対象外、検討記録)

- **トランスポートアダプタ(無線ペア / TCP)**: core への新要求はなし(アダプタ境界で完結)。設計メモ: 継続系はスナップショット + ハートビート(冪等・自己回復)、単発系はシーケンス番号 + ACK/再送の 2 チャネル。タイムアウト = 切断(仮置き 6 適用)。暗号化・ペア固定は必須要件。ワイヤ形式にバージョン。
- **出力切替(KVM 的)**: アクティブ出力 1 変数 + 切替時に旧出力へ空スナップショット・新出力へ現在集合。正本はアクティブに追随。切替時 lock 同期(NumLock tap 等)はオプション候補。現実の HID 出力多重度は実質 2(USB+BLE。P4 なら USB×2 の可能性)。
- **能動 lock 同期**: ズレたホストへ toggle 打鍵を注入して収束させる方式。発振対策(整定ウィンドウ)、リトライ上限、CapsLock はホストプロファイル必須(ja_jp Windows は Shift+CapsLock、macOS は長押し)という制約ごと記録。
- **tap/hold、core 汎用トグル / ワンショット変換**: 時間依存。必要になったユースケースで再検討。
- **絶対座標(タブレット・タッチ)、AltGr を使う layout、ポインタ加速**。
