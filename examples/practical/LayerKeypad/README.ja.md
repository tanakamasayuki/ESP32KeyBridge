# LayerKeypad

自作の小型キーパッドに「1 キーで 2 機能」を持たせたいとき用。4 キーが**通常はドキュメント/スライド操作**(Page Up/Down・Home・End)、**Fn キーを押している間だけメディアキー**(音量・曲送り)になります。物理パッドは 1 つ、機能は 2 面 — ノート PC の Fn レイヤーと同じ考え方を、ただのボタンで作ります。

- **レイヤーの仕組み**: Fn キーは仮想キー(`VirtualUsage::V1`)で、レイヤーのトリガ専用です(PC には届きません)。Fn を押している間だけ `layer.remap()` の差し替えが効き、Fn を離せば各キーは基層(base)の機能に戻ります。
- **キー種別をまたぐ**: レイヤーの remap は種別をまたげます。この例では keyboard キー(PageUp など)を consumer キー(音量など)に差し替えています。
- **押下時に確定**: キーは押した瞬間に機能が決まり、離すまで保持されます。キーを押したまま Fn を離しても、そのキーは押したときの機能のままです(スタックしません)。

## キー割り当て

| ピン | 通常(base) | Fn 長押し中 |
|---|---|---|
| GPIO4 | **Fn**(レイヤーキー) | — |
| GPIO5 | Page Up | 音量 + |
| GPIO6 | Page Down | 音量 − |
| GPIO7 | Home | 前の曲 |
| GPIO8 | End | 次の曲 |

キーを足す・割り当てを変えるには、`addKey()` と `fn.remap()` を増やすだけです。レイヤーを重ねたい場合は Fn を別の仮想キーにして `addLayer()` をもう 1 枚足します(`MaxLayers` まで)。

## ハードウェア構成(ESP32-S3)

- ESP32-S3 の USB OTG ポートを PC へ挿します(キーボードを含む複合 HID デバイスとして見えます)。
- モーメンタリスイッチを各ピンと GND の間に接続します(内部プルアップ、閉 = 押下)。配線に合わせてピン番号を変えてください。

> **注意**: 使用するアダプタ(GPIO キー入力 / USB Device 複合 HID 出力)は実装済みです(実機検証はこれから)。
