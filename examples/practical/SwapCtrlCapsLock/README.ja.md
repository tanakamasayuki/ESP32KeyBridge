# SwapCtrlCapsLock

キーボードと PC の間に挟んで、CapsLock と Ctrl を入れ替えたいとき用。PC にもキーボードにも設定を入れません。

- remap は 1 段の lookup で連鎖しないため、2 エントリの登録がそのまま「入れ替え」になります。長押し・同時押し・ホスト側オートリピートは保たれます。
- 入れ替え後の Ctrl 位置で Caps を切り替えても、PC からの LED report を転送するのでキーボードの LED は正しく追随します。

## ハードウェア構成(ESP32-P4)

USB を 2 系統(Host + Device)使うため ESP32-P4 が必要です。P4 の USB は 3 つあります:

| ポート | ピン | この example での役割 |
|---|---|---|
| HS(High-Speed) | - | **USB Device(PC へ挿す)**。arduino-esp32 3.3.10 のライブラリ制約で Device は HS 固定 |
| FS CDC | GPIO24/25 | ボードによっては USB Host に使う(下記スワップ) |
| FS OTG | GPIO26/27 | **USB Host(キーボードを挿す)**の既定 |

このほかにボードによっては外部 USB シリアル変換チップのポートがあります。どのポートがコネクタに配線されているかはボードごとに違うので、まず確認してください。

### FS ポートのスワップ

USB Host は FS 側を明示指定しています(`ESP_USB_HOST_PORT_FULL_SPEED`)。キーボードを挿すコネクタが FS の CDC ピン(GPIO24/25)側に配線されているボードでは、**USB 初期化前に** CDC と OTG の PHY を入れ替えます。スケッチ内の次の行のコメントアウトを外してください:

```cpp
usb_wrap_ll_phy_select(&USB_WRAP, 0);
```

なお、**ボードの表記と実際の配線が一致しないことがあります**。表記だけで判断せず、コネクタがどのピン(OTG = GPIO26/27 / CDC = GPIO24/25)に配線されているかを確認してください。

### 事例: M5Stack Tab5

- Type-A = HS = **USB Device**。PC へ接続する側です(ホスト用に見えますが逆)。
- Type-C = **表記は OTG ですが実際は FS CDC** = USB Host。キーボードはこちらに接続し、上記の PHY スワップが必要です。表記と実配線が違う例です。

### HS 側を USB Host に使う場合の制約(参考)

USB Device を FS で使えるようにする変更は arduino-esp32 へプルリクエスト提出中で、3.3.10 時点では未マージです(そのため本 example は Device = HS 固定)。マージされれば Host を HS 側にする構成も可能になりますが、その場合は次の制約があります:

- **HS ポートの Host は HS / FS デバイスの混在に対応しません。** HS 対応 USB ハブの先に FS デバイス(一般的な USB キーボード等)をつないでも動作しません。
- したがって HS 側を Host にする場合は、**ハブを使わず 1 デバイスのみを直結**してください。

> **注意**: USB Device 出力アダプタ(複合 HID)は実装済み(実機検証はこれから)です。USB Host 入力アダプタはビルド確認用のモックです(実装順 7 で実装されます)。
