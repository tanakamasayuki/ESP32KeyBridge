# NumPad

自作テンキー・マクロパッド・自作キーボードを作りたいとき用。スイッチをマトリクス配線(各スイッチを行線と列線の交点に置く)して GPIO に直結し、USB キーボードとして PC につなぎます。この例は 4×3 = 12 キーのテンキーです。

- 配線とキーマップは分離しています。`setRowPins()` / `setColPins()` が配線(行 = スキャン出力、列 = 内部プルアップ入力)、`setKeys()` がキーマップで、**行優先の並び順 = 物理配置**です(コードの見た目がそのままキー配置になります。QMK のキーマップと同じ慣習)。座標や index は登場しません。
- 使わない位置には無効キー `esp32keybridge::Key()` を置きます。要素数が 行×列 と合わない場合は `setKeys()` が false を返します。
- スキャンとデバウンス(`setDebounceMillis()`)はアダプタ内です。
- **ダイオードは任意**です。無しの場合、3 キー以上の同時押しで発生するゴースト(押していないキーが押されたように見える現象)をアダプタがブロックするため、確実な同時押しは 2 キーまでになります。全スイッチにダイオードを入れた場合は `pad.setHasDiodes(true)` でブロックを外せます(同時押し無制限)。
- Keypad の数字キーは **PC 側の NumLock 状態**で数字/カーソル移動が切り替わります(実物のテンキーと同じ)。lock 状態の正本は PC です。

## 応用: マクロパッドにする

キーを `KeyboardUsage::F13`〜`F24`(物理キーボードにほぼ存在しない拡張ファンクションキー)にすると、アプリ側のショートカット割り当て専用パッドになります。virtual キー + `config.textMacro(...)` を割り当てれば定型文パッドです(FootSwitch の例を参照)。

## ハードウェア構成(ESP32-S3)

- ESP32-S3 の USB OTG ポートを PC へ挿します(キーボードを含む複合 HID デバイスとして見えます)。
- スイッチは行線と列線の交点に接続します。この例の配線: 行 = GPIO4/5/6/7、列 = GPIO10/11/12(配線に合わせて変更してください)。

> **注意**: `GpioMatrixInputAdapter` は現在ビルド確認用のモックです(API 確認中。実装はこれから)。
