# ScrollDial

手元に物理的なスクロールダイヤルが欲しいとき用。ロータリーエンコーダを回すとマウスホイールがスクロールし、軸を押し込むと中クリックになります。PC からは普通の USB マウス(ホイール + ボタン)に見えるので、ドライバも設定も不要です。

- VolumeKnob が `mapToKeys()`(メディアキーを叩く)なのに対し、これは `mapToAxis()` で **1 デテントを相対軸に流す**版です。エンコーダはハードウェア計数(PCNT)なので、`loop` が詰まっても回転を取りこぼしません(反映が遅れるだけ)。
- 押し込みスイッチはただの GPIO キーなので `GpioKeyInputAdapter` を併用し、マウスの中ボタンに割り当てています。ダイヤルとスイッチは 2 つの入力として 1 つのマウスに統合されます。
- スクロールの**速さと向き**は `config.setAxisScale(Axis::Wheel, n)` で決めます。負値で反転(macOS の「ナチュラル」スクロール相当)、絶対値を大きくすると速くなります。

## 応用: ジョグダイヤルにする

`Axis::Wheel` の代わりに `Axis::X` へ流すと、動画編集のジョグ(スクラブ)や水平スクロールに使えます:

```cpp
dial.mapToAxis(esp32keybridge::Axis::X);
```

1 デテントあたりのパルス数(既定 4 = EC11)や x4 直交デコードの詳細は [../../../docs/ADAPTERS.ja.md](../../../docs/ADAPTERS.ja.md) の「ロータリーエンコーダ: 直交デコードとデテント」を参照してください。

## ハードウェア構成(ESP32-S3)

- ESP32-S3 の USB OTG ポートを PC へ挿します(マウスを含む複合 HID デバイスとして見えます)。
- EC11 系エンコーダ: A 相 = GPIO5、B 相 = GPIO6、コモン = GND、押し込みスイッチ = GPIO7 と GND(配線に合わせて変更してください)。

> **注意**: 使用するアダプタ(ロータリーエンコーダ入力 / GPIO 入力 / USB Device 複合 HID 出力)は実装済みです(実機検証はこれから)。
