# VolumeKnob

手元に物理的な音量つまみが欲しいとき用。ロータリーエンコーダを回すと PC の音量が上下し、押すとミュートします。USB のメディアキーとして動くので、PC 側の設定・常駐ソフトは不要です。

- エンコーダは PCNT ペリフェラルでハードウェア計数します(割り込みなし)。loop が長時間ブロックしても回転はカウンタに溜まっているので取りこぼしません(反映が遅れるだけ)。1 ノッチ(デテント)= 1 タップで、`mapToKeys(時計回りのキー, 反時計回りのキー)` に割り当てます。
- エンコーダの押し込みスイッチはただの GPIO キーなので、`GpioKeyInputAdapter` を併用します(アダプタは何個でも足せます)。
- 内部は A・B 両相・両エッジのフル x4 直交デコードなので、1 直交周期で 4 カウント出ます。それを `setPulsesPerDetent`(既定 4 = EC11 の 1 ノッチ = 全周期)で割って「1 ノッチ = 1 タップ/±1」に正規化します(余りは持ち越すのでドリフトなし)。1 ノッチで 2 出る/半分しか出ない型は `knob.setPulsesPerDetent(n)` で合わせます。詳細は [../../../docs/ADAPTERS.ja.md](../../../docs/ADAPTERS.ja.md) の「ロータリーエンコーダ: 直交デコードとデテント」。
- エンコーダを増やす場合はアダプタを 1 つずつ追加します(PCNT のユニット数まで。ESP32-S3 は 4 ユニット)。

## 応用: スクロールダイヤル・ジョグにする

タップの代わりに相対軸へ流すと、ホイールやポインタとして使えます:

```cpp
knob.mapToAxis(esp32keybridge::Axis::Wheel);
```

向きと速さは設定側で変えられます(`config.setAxisScale(Axis::Wheel, -2)` = 反転 + 2 倍。NaturalScroll の例を参照)。動画編集のジョグ(`Axis::X`)にも同じ形で使えます。

## ハードウェア構成(ESP32-S3)

- ESP32-S3 の USB OTG ポートを PC へ挿します(キーボードを含む複合 HID デバイスとして見えます)。
- EC11 系エンコーダ: A 相 = GPIO5、B 相 = GPIO6、コモン = GND、押し込みスイッチ = GPIO7 と GND(配線に合わせて変更してください)。

> **注意**: 使用するアダプタ(ロータリーエンコーダ入力 / GPIO 入力 / USB Device 複合 HID 出力)は実装済みです(実機検証はこれから)。
