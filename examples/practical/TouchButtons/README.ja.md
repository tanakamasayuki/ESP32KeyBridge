# TouchButtons

**自作インプットアダプタの書き方**を示す example です。題材として ESP32-S3 内蔵の静電容量タッチを使い、パッドに触れると PC にメディアキーが送られます。外部ライブラリも追加部品も不要(タッチ GPIO に短い導線・箔・あるいはピンそのままで反応)。

I2C エキスパンダ、各種センサ、ネットワーク受信など「入力になり得るもの」は無数にあります。すべてをこのライブラリに同梱する代わりに、**自分でアダプタを書けるように**この 1 本を用意しています。タッチはその題材にすぎません。

## アダプタの契約(これだけ)

`esp32keybridge::InputAdapter` を継承し、次を実装します。

- **`update()`(必須)**: ハードを読み、内部の `KeySet` を `press()` / `release()` で更新する。`bridge.update()` ごとに 1 回呼ばれます。
- **`keys()`(必須)**: 「今押されているキー集合」を返す。ブリッジが読むのはこれだけ。

任意のオーバーライド:

- `connected()`(既定: 常に接続)— 切断を報告すると、そのアダプタのキーがマージから外れます(スタック防止)。
- `setLockState(state)`(既定: 無視)— ホストの Lock 状態(CapsLock 等)を自分のデバイスへ反映したいとき(LED など)。
- `takeAxisDelta(axis)`(既定: 0)— マウス移動・ホイール・エンコーダのような相対量を出したいとき。

`update()` で好きなハード(I2C・センサ・ネットワーク)を読み、`KeySet` に押下を反映し、`keys()` で返す。あとの統合・remap・レイヤー・出力はブリッジがやります。この example の `TouchButtons` クラスがその最小テンプレートです。

## この題材でのポイント

- **遅延初期化**: 無タッチ基準値の測定は、コンストラクタではなく最初の `update()` で行います(ペリフェラルが確実に準備済み。GPIO / エンコーダ系アダプタと同じ流儀)。
- **ESP32-S3 の `touchRead()` はタッチで値が“増える”**(旧 ESP32 は逆)。押下は基準値より十分高いとき、解放は基準値寄りで、というヒステリシスでチャタリングを防いでいます。効き具合は係数を調整(`value` を print すれば生値を確認できます)。
- **タッチ対応 GPIO**: ESP32-S3 は GPIO1〜14。配線に合わせて変更してください。

## キー割り当て(初期値)

| ピン | キー |
|---|---|
| GPIO4 | 音量 − |
| GPIO5 | 再生/停止 |
| GPIO6 | 音量 + |

`addPad(pin, key)` の `key` は種別自由(keyboard / consumer / mouse ボタン / 仮想キー)。仮想キーにして config 側で remap・レイヤーに載せることもできます。

## ハードウェア構成(ESP32-S3)

- ESP32-S3 の USB OTG ポートを PC へ挿します(メディアキーを含む複合 HID デバイスとして見えます)。
- タッチ GPIO(4/5/6)に導線や箔を付けるとタッチ面積を稼げます。

> **注意**: 使用するアダプタ(自作タッチ入力 / USB Device 複合 HID 出力)のうち、`TouchButtons` はこの example 内に実装しています。USB Device 出力は実装済みです(実機検証はこれから)。
