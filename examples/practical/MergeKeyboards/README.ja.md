# MergeKeyboards

複数の入力デバイスを 1 つのキーボードとして PC に見せたいとき用。USB キーボードと GPIO の追加ボタンを統合し、複合 HID デバイスとして PC へ送ります。どちらの入力で押したキーもまとまって出るので、既存のキーボードに物理キーを増設する感覚で使えます。

## 2 種類の統合

- **USB Host キーボードアダプタは、スタック上の全キーボードを自動で統合**します(USB ハブに複数キーボードを挿すと、最大 4 台までまとまります)。これは `addInput()` 1 回です。
- **別種の入力(ここでは GPIO マクロパッド)をもう 1 つ `addInput()`** すると、アダプタをまたいで統合されます。

統合は**和集合**です。あるキーは、それを押しているすべての入力が離すまで押下扱いのまま。2 つのデバイスで同じキーを押しても、片方を離した瞬間に離れてしまう(スタックする)ことはありません。

## キー割り当て(GPIO マクロパッド)

| ピン | キー |
|---|---|
| GPIO4 | ミュート |
| GPIO5 | 再生/停止 |
| GPIO6 | F13(アプリのショートカット割り当て用) |

`macroPad.addKey()` を増やせばボタンを追加できます。キー種別は自由(keyboard / consumer / mouse ボタンなど)です。

## ハードウェア構成(ESP32-P4)

USB を 2 系統(USB キーボード入力 + USB Device 出力)使うため ESP32-P4 が必要です。**ESP32-S3 など USB 1 系統のチップでは、USB キーボード入力と USB Device 出力を同時に持てないため単体では動きません**(GPIO だけを統合するなら S3 でも可能です)。

- **USB Device = HS ポート**: PC へ挿します。
- **USB Host = FS ポート**: キーボード(または複数キーボードを挿したハブ)を挿します。
- GPIO マクロボタン: 各ピンと GND の間にモーメンタリスイッチ(内部プルアップ、閉 = 押下)。

P4 の USB ポートの詳細(コネクタとピンの対応、FS PHY スワップ、M5Stack Tab5 の事例)は [../SwapCtrlCapsLock/README.ja.md](../SwapCtrlCapsLock/README.ja.md) が基準です。

> **注意**: 使用するアダプタ(USB Host キーボード入力 / GPIO キー入力 / USB Device 複合 HID 出力)は実装済みです(実機検証はこれから)。USB Host 入力には EspUsbHost 2.3.0 以上が必要です。
